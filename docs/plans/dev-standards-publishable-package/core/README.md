# Core

The `core` directory should contain the universal rules of the standard.

These rules are always active and should remain valid regardless of:

- project topic
- stack
- architecture
- team shape
- toolchain

## Initial Core Catalog

1. Scope And Applicability
2. Repository Contract
3. Configuration And Environment
4. Change Management
5. Decision Records
6. Problem Tracking
7. Dependency Management
8. Verification
9. Interfaces And Contracts
10. Observability
11. Security And Risk
12. Documentation And Discoverability

## Authoring Standard

Each core rule should answer:

- what the rule requires
- why it exists
- what minimum evidence or behavior is expected
- what it does not prescribe
